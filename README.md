# Telegram Monitor - Мониторинг чатов отдела продаж

Скрипт для автоматического мониторинга Telegram аккаунта отдела продаж с целью контроля своевременности ответов клиентам.

## Описание

Скрипт анализирует все чаты общего Telegram аккаунта компании. В зависимости от режима (`REPORT_TYPES`) он:

- **`unread`** — выявляет непрочитанные сообщения от клиентов
- **`unanswered`** — выявляет неотвеченные сообщения
- **`leaderboard`** — строит топ переписок по среднему времени ответа менеджера

После анализа скрипт отправляет сводку указанному пользователю (по умолчанию `@victorryakh`).

## Особенности

✅ **Учет рабочих часов** - скрипт считает только рабочее время (9:00-21:00 по умолчанию)
✅ **Реакции как ответ** - проставленная реакция на сообщение клиента считается ответом
✅ **Исключение чатов** - возможность исключить определенные чаты из мониторинга
✅ **Кеширование сессии** - не требуется повторная авторизация при каждом запуске
✅ **Фильтрация каналов** - каналы не учитываются, только личные чаты и группы
✅ **Docker Compose** - готовый к запуску контейнер с bind mount сессии
✅ **Топ по скорости ответа** - рейтинг переписок с наименьшим средним временем ответа

## Требования

- Docker и Docker Compose
- Telegram API credentials (API ID и API Hash)
- Доступ к Telegram аккаунту (номер телефона)

## Установка

### 1. Клонирование репозитория

```bash
git clone <repository-url>
cd telegram-police
```

### 2. Получение Telegram API credentials

1. Перейдите на <https://my.telegram.org/auth>
2. Войдите с номером телефона вашего аккаунта
3. Перейдите в раздел "API development tools"
4. Создайте новое приложение (если еще не создано)
5. Скопируйте **API ID** и **API Hash**

### 3. Настройка `.env`

Создайте файл `.env` в корне проекта (или скопируйте из примера):

```bash
cp env.example .env
```

Заполните обязательные переменные:

```env
API_ID=12345678
API_HASH=your_api_hash_here
PHONE=+79991234567
```

Полный список переменных с описанием — в разделе [Конфигурация](#конфигурация).

### 4. Первичная авторизация

При первом запуске Telegram запросит код подтверждения. Это нужно сделать интерактивно:

```bash
docker compose run --rm telegram-police
```

Вы получите:

1. Код в Telegram (от Telegram)
2. Запрос на ввод кода в терминале
3. Возможно, запрос пароля двухфакторной аутентификации (если включена)

После успешной авторизации в корне проекта создастся файл `session.session`. Он монтируется в контейнер через bind mount и сохраняется между перезапусками.

## Использование

### Запуск через Docker Compose

```bash
# Собрать образ и запустить
docker compose up --build

# Запустить в фоне
docker compose up -d

# Однократный запуск (без restart: unless-stopped)
docker compose run --rm telegram-police
```

### Просмотр логов контейнера

```bash
docker compose logs -f
```

### Остановка

```bash
docker compose down
```

### Настройка автоматического запуска через cron (на хосте)

```bash
crontab -e

# Запуск каждый час с 9:00 до 21:00
0 9-21 * * * cd /path/to/telegram-police && docker compose run --rm telegram-police
```

## Конфигурация

Все параметры задаются через файл `.env`. Переменные с пометкой **обязательно** не имеют значения по умолчанию.

| Переменная | Обязательно | По умолчанию | Описание |
| --- | --- | --- | --- |
| `API_ID` | ✅ | — | Telegram API ID |
| `API_HASH` | ✅ | — | Telegram API Hash |
| `PHONE` | ✅ | — | Номер телефона аккаунта |
| `REPORT_TO` | | `@victorryakh` | Username получателя сводки |
| `WORKING_HOURS_START` | | `9` | Начало рабочего дня (часы) |
| `WORKING_HOURS_END` | | `21` | Конец рабочего дня (часы) |
| `EXCLUDED_CHATS` | | `["@PremiumBot","@SpamBot"]` | JSON-список исключённых чатов |
| `REPORT_TYPES` | | `unread,unanswered` | Типы отчётов через запятую: `unread`, `unanswered`, `leaderboard` |
| `LEADERBOARD_RESPONSE_LIST_COUNT` | | `10` | Количество чатов в топе (для `leaderboard`) |
| `LEADERBOARD_RESPONSE_DAYS` | | `7` | Глубина анализа в днях (для `leaderboard`) |

### Примеры `EXCLUDED_CHATS`

```env
# По username
EXCLUDED_CHATS=["@spambot","@notifications"]

# По названию группы
EXCLUDED_CHATS=["Внутренний чат","Spam Group"]

# Смешанный список
EXCLUDED_CHATS=["@bot","Внутренний чат"]
```

## Логика работы

### Рабочие часы

Скрипт учитывает только **рабочие часы** (по умолчанию 9:00-21:00):

**Пример 1:** Клиент написал в 03:00, сейчас 10:00

- Рабочее время началось в 09:00
- Прошел 1 рабочий час (с 09:00 до 10:00)
- ❌ НЕ добавляем в сводку (< 2 рабочих часов)

**Пример 2:** Клиент написал в 03:00, сейчас 12:00

- Рабочее время началось в 09:00
- Прошло 3 рабочих часа (с 09:00 до 12:00)
- ✅ Добавляем в сводку (>= 2 рабочих часов)

**Пример 3:** Клиент написал в 20:00 (вчера), сейчас 10:00 (сегодня)

- Вчера: 1 рабочий час (20:00-21:00)
- Сегодня: 1 рабочий час (09:00-10:00)
- Итого: 2 рабочих часа
- ✅ Добавляем в сводку (>= 2 рабочих часов)

### Реакции как ответ

Если на сообщение клиента была поставлена реакция от вашего аккаунта, это считается ответом, и сообщение НЕ попадает в сводку неотвеченных.

### Топ по скорости ответа (`leaderboard`)

Для каждой переписки берутся сообщения за последние `LEADERBOARD_RESPONSE_DAYS` дней. По каждой паре «сообщение клиента → первый ответ менеджера» считается рабочее время ответа. Итоговый показатель — среднее по всем парам. В отчёт попадают `LEADERBOARD_RESPONSE_LIST_COUNT` переписок с наименьшим средним временем ответа.

## Форматы сводки

### `unread` / `all` — если проблем нет

```text
✅ Сводка по чатам (07.05.2026 12:00)

Все чаты обработаны. Проблем не обнаружено.
```

### `unread` / `all` — если есть проблемы

```text

📬 НЕПРОЧИТАННЫЕ СООБЩЕНИЯ (2):
🔵 @client1 (Иван Иванов) - 3 непрочитанных, первое от 07.05.2026 09:30
```

## Устранение неполадок

### Повторная авторизация

```bash
rm session.session
docker compose run --rm telegram-police
```

### Скрипт не находит непрочитанные сообщения

Убедитесь, что:

1. Сообщения действительно непрочитаны в Telegram
2. Чат не находится в `EXCLUDED_CHATS`
3. Это личный чат или группа (не канал)

### Скрипт не отправляет сводку

Проверьте:

1. Правильность `REPORT_TO` в `.env` (должен начинаться с `@`)
2. Наличие диалога с этим пользователем в Telegram
3. Логи: `docker compose logs`

## Безопасность

- Файл `.env` содержит credentials — не коммитьте его (уже в `.gitignore`)
- Файл `session.session` содержит данные авторизации — не передавайте его третьим лицам (уже в `.gitignore`)
- Храните API_ID и API_HASH в безопасности
